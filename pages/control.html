What each term does

P — Proportional

Force ∝ current error. Pull harder when far away. High Kp → fast response but overshoot & oscillation. Low Kp → slow, often steady-state error.

I — Integral

Force ∝ accumulated error over time. Kills steady-state offset (e.g. friction holding the cart back). Too much Ki → wind-up & slow oscillation.

D — Derivative

Force ∝ rate of change of error. Acts like a brake when approaching the target. Damps overshoot & oscillation, but amplifies sensor noise.

u(t) = Kp·e(t) + Ki·∫e dt + Kd·(de/dt)
e(t) = setpoint − position

A worked PID step

Take Kp = 2, Ki = 0.5, Kd = 1, sampled every dt = 0.5 s. The error shrinks each step: e = 4, 3, 2. Watch how each term behaves as the cart closes in.

k=0  e = 4
     integral = 0 + 4·0.5 = 2.0     deriv = (4 − 0)/0.5 = +8.0
     P = 2·4   = 8.00     I = 0.5·2.0 = 1.00     D = 1·(+8.0) = +8.00
     u = P + I + D = 8.00 + 1.00 + 8.00 = 17.00

k=1  e = 3
     integral = 2.0 + 3·0.5 = 3.5   deriv = (3 − 4)/0.5 = −2.0
     P = 2·3   = 6.00     I = 0.5·3.5 = 1.75     D = 1·(−2.0) = −2.00
     u = 6.00 + 1.75 − 2.00 = 5.75

k=2  e = 2
     integral = 3.5 + 2·0.5 = 4.5   deriv = (2 − 3)/0.5 = −2.0
     P = 2·2   = 4.00     I = 0.5·4.5 = 2.25     D = 1·(−2.0) = −2.00
     u = 4.00 + 2.25 − 2.00 = 4.25

Notice the pattern: P falls as the error shrinks; I keeps growing because it accumulates every past error; and D flips negative once the error starts decreasing, acting as a brake against overshoot. The live sim above runs exactly this loop ~60 times a second.

Open loop vs. closed loop

Open loop

Command an action and hope. "Run the motor at 70% for 2 seconds." No sensor feedback, so any disturbance — a ramp, a heavier load, low battery — throws it off. Simple, but it can't correct itself.

Closed loop

Measure the output, compute the error, and feed it back — exactly what PID does. The cart reaches the setpoint despite kicks and friction because every cycle re-checks reality. Click "⚡ Kick the cart" above to see the loop fight back.

Drives, motors & gearboxes — a torque/speed trade

Control output eventually drives a motor through a gearbox. A gear ratio N:1 divides speed by N and multiplies torque by N (minus a few percent friction). Take a motor spinning 300 RPM at 5 N·cm through a 48:1 reduction:

output speed  = 300 / 48      = 6.25 RPM
output torque = 5 · 48        = 240 N·cm   (ideal)
              = 5 · 48 · 0.92 = 220.8 N·cm  (at ~92% efficiency)

Power is conserved (ideally): motor side (0.05 N·m)·(31.42 rad/s) = 1.57 W equals output side (2.40 N·m)·(0.654 rad/s) = 1.57 W. The gearbox trades the fast-but-weak motor for the slow-but-strong wheel a Pi car needs.

Tuning hint — Start with Kp only (set Ki = Kd = 0). Increase Kp until you see oscillation, then back off ~30%. Add Kd to damp the oscillation. Add a little Ki last to eliminate any remaining offset. This is the classic Ziegler-Nichols heuristic.
