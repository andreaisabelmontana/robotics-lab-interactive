The kinematics, exactly

A differential-drive robot has two independently driven wheels separated by a wheelbase b. From the two wheel speeds you recover the robot's body twist — how fast it moves forward and how fast it rotates:

v = (v_R + v_L) / 2          ← linear speed of the body centre
ω = (v_R − v_L) / b          ← angular speed (rad/s)

Then you integrate that twist forward over a time step dt to update the pose (x, y, θ). When the robot turns (ω ≠ 0) it traces a perfect circular arc of radius R = v/ω, and the exact closed-form update is:

R = v / ω
x ← x + R·( sin(θ + ω·dt) − sin θ )
y ← y − R·( cos(θ + ω·dt) − cos θ )
θ ← θ + ω·dt

When ω ≈ 0 the arc radius blows up, so we fall back to the straight-line form (the limit of the equations above):

x ← x + v·cos(θ)·dt
y ← y + v·sin(θ)·dt

This is the same math the sim above runs ~60 times a second. The three special cases you can dial in with the presets:

vL = vR

ω = 0 → drives in a straight line along the current heading. Distance covered is exactly v·dt.

vL = −vR

v = 0, ω ≠ 0 → spins in place. The body centre never moves; only θ changes.

one wheel stopped

Say vL = 0. The robot pivots about that wheel, tracing an arc of radius exactly R = b/2.

A worked step — from encoder ticks to a pose update

On the real car you don't read wheel speeds directly; you count encoder ticks and convert. With a 20-slot encoder, a wheel radius of 3 cm (circumference 18.85 cm) and a wheelbase b = 15 cm, suppose one cycle records 40 ticks left and 44 ticks right, starting from heading 30°:

d_L = (40 / 20)·18.85 = 37.70 cm        d_R = (44 / 20)·18.85 = 41.47 cm
d_center = (d_L + d_R) / 2 = 39.58 cm      ← how far the centre moved
Δθ       = (d_R − d_L) / b = (41.47 − 37.70) / 15 = 0.2513 rad = 14.40°

θ_mid = 30° + Δθ/2 = 37.20°              ← integrate about the mid-heading
Δx = d_center·cos(θ_mid) = 39.58·cos(37.20°) = 31.53 cm
Δy = d_center·sin(θ_mid) = 39.58·sin(37.20°) = 23.93 cm
new heading = 30° + 14.40° = 44.40°

That midpoint formula is the second-order approximation taught in most courses. The sim above uses the exact arc integral instead, which lands a hair shorter (the chord of the arc, ≈ 31.45 cm / 23.87 cm) but recovers the identical Δθ = 14.40° and new heading 44.40°. For one short step the two agree to fractions of a millimetre.

Dead-reckoning drifts. Every cycle's small error in wheel diameter, tick count, or wheel slip accumulates — there's no feedback to correct it, so the estimated pose slowly diverges from reality. That's why real robots fuse odometry with another sensor (IMU, ultrasonic landmarks, a camera) rather than trusting wheel counts alone. Run the sim long enough and you're trusting exactly this chain of integrations.
